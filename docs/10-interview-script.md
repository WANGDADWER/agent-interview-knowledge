# 两分钟项目介绍模板

我做的是一个面向企业知识库/业务流程的 AI Agent 系统。它不是简单聊天机器人，而是由 RAG 数据层、Agent 决策层、图式编排层和安全观测层组成。数据层负责文档解析、切块、向量化、混合检索和 rerank；Agent 层根据用户意图选择检索、数据库查询或业务 API；编排层用图工作流管理状态、重试、人工审批和失败恢复；生产层关注权限控制、Prompt Injection 防护、trace、离线评测和成本延迟优化。最终我用任务成功率、检索召回、答案忠实性、工具调用正确率、延迟和 token 成本来评估系统。

# 面试前最后检查清单

- 能用 1 分钟解释 Agent、RAG、Workflow、多 Agent 的区别。
- 能画出企业知识库 Agent 的系统架构图，并解释每一层。
- 能说明 LangChain、LangGraph、LlamaIndex、AutoGen/MAF 的定位差异。
- 能讲清楚一次错误回答如何从 trace 中定位。
- 能讲清楚权限、审批、审计、沙箱和 Prompt Injection 防护。
- 准备 2 个真实失败案例：召回失败、工具误调用、成本过高或多 Agent 跑偏，并说明修复方案。
