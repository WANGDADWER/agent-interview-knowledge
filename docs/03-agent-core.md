# 一、Agent 基础与核心原理

## Q1：什么是 AI Agent？它和普通 Chatbot 有什么区别？

**候选人回答：** AI Agent 是围绕目标进行感知、决策和行动的软件实体。普通 Chatbot 主要生成回复；Agent 还会维护状态、选择工具、调用外部系统、观察结果并继续决策。面试时不要只说“LLM + Tool”，更完整的表达是：Agent = LLM/策略模块 + 状态/记忆 + 工具 + 环境反馈 + 控制流 + 安全约束。

**追问：** RAG 是 Agent 吗？答：RAG 本身通常是上下文增强流程；当系统能基于目标自主选择检索、调用工具、反思和执行多步动作时，才更接近 Agent。

## Q2：Agent 的主循环是什么？

**候选人回答：** 常见主循环是 Observe → Plan/Reason → Act → Observe → Stop/Reflect。工程上要把它实现成可控状态机：输入消息、当前状态、可用工具、模型决策、工具结果、错误处理、终止条件都要显式管理。

**追问：** 为什么不能无限循环？答：需要 max_steps、超时、预算、置信度阈值、任务完成判定、人工接管等机制。

## Q3：Tool Calling、Function Calling 和 MCP 的区别是什么？

**候选人回答：** Function calling 通常指模型 API 以结构化方式返回函数调用意图；Tool 是业务侧真正暴露给模型的能力，如查库、发邮件、执行代码；MCP 是把工具、资源等能力标准化暴露给模型/Agent 的协议层。面试回答要强调：模型只是提出调用，真正执行必须经过权限、参数校验、日志和必要的人类确认。

**追问：** 模型输出了一个危险调用怎么办？答：使用工具白名单、schema 校验、权限分级、审批流、沙箱和审计日志。

## Q4：Agent 为什么需要 Memory？Memory 分哪些类型？

**候选人回答：** Memory 不是越多越好，而是为了解决短上下文、多轮一致性和个性化问题。常见分法：短期记忆保存当前任务状态；长期记忆保存跨会话偏好或事实；情节记忆保存历史交互；语义记忆保存知识；工作记忆用于当前推理步骤。

**追问：** Memory 可能带来什么风险？答：错误记忆、隐私泄露、过期信息、检索噪声和成本增加，因此需要写入规则、更新策略、遗忘机制和权限控制。

## Q5：Planning 在 Agent 中有什么作用？

**候选人回答：** Planning 用来把复杂目标拆成可执行步骤。简单任务可以直接工具调用；复杂任务可以使用 task decomposition、plan-and-execute、planner-executor、self-reflection 或 graph workflow。工程上，规划结果最好结构化，便于校验、重试和可视化。

**追问：** 为什么规划不一定越复杂越好？答：规划会增加 token、延迟和错误传播；确定性流程优先用代码/图编排，开放性任务再交给 LLM 规划。

## Q6：如何评价一个 Agent 是否“好”？

**候选人回答：** 不要只看最终答案。评价维度包括任务成功率、工具调用正确率、步骤数、平均延迟、token 成本、幻觉率、可追溯性、安全违规率、人工介入率和用户满意度。大厂面试更看重可复现的离线评测集与线上监控闭环。

**追问：** 如何做离线评测？答：构建标准任务集，标注期望结果、允许工具调用轨迹，分别评估 answer correctness、faithfulness、retrieval recall、tool precision 和 safety。

## Q7：Agent 和传统工作流系统的边界在哪里？

**候选人回答：** 如果流程稳定、规则明确、输入结构化，传统 workflow 更可靠；如果任务开放、信息不完整、需要自然语言理解和动态工具选择，Agent 更合适。成熟系统通常是“确定性工作流 + 局部 LLM/Agent 决策”的混合架构。

**追问：** 大厂落地为什么喜欢混合架构？答：可控、可审计、可降级，成本与风险更可管理。
