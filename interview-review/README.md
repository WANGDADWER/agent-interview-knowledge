# 面试复盘与追问准备

这个目录用于把源码精读和实战项目转化为面试表达。

## 建议内容

- `source-code-questions.md`：源码追问题；
- `system-design-review.md`：系统设计题复盘；
- `project-pitch.md`：项目两分钟介绍；
- `failure-cases.md`：失败案例与修复方案。

## 复盘原则

每次学习后，都要把内容转化为三种表达：

```text
一句话解释
→ 三分钟源码讲解
→ 十分钟系统设计讲解
```

真正精通不是知道概念，而是能在面试中把“官方定位、源码机制、工程落地、项目结果”连起来讲。
