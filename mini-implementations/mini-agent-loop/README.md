# mini-agent-loop：手写最小 Agent 执行循环

## 目标

理解 Agent 的最小组成：

```text
用户目标
→ 模型/策略决定下一步
→ 调用工具
→ 观察工具结果
→ 更新状态
→ 继续或停止
```

## 运行方式

```bash
python mini_agent_loop.py
```

## 这个 mini 版本实现了什么？

- 工具注册；
- 简单策略路由；
- 工具调用；
- 状态记录；
- 最大步数限制；
- 最终回答。

## 它没有实现什么？

- 没有真实 LLM 调用；
- 没有 function calling schema；
- 没有 tracing 平台；
- 没有并发、流式输出、权限系统；
- 没有真实 LangChain 源码逻辑。

## 面试表达

这个 mini 版本说明我理解 Agent loop 的本质：Agent 不是一次模型调用，而是围绕任务状态不断选择动作、调用工具、观察结果并判断是否完成。生产系统还需要权限控制、失败恢复、trace、成本控制和安全审批。
